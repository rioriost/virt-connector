# VirtConnector

VirtConnector is a macOS menu bar utility that mirrors Mac power events to user-configured Matter-capable devices.

## Goals

- Multiple devices can be registered.
- Each device can be independently enabled or disabled.
- Each device can choose No Change, On, or Off for Wake, Sleep, and Power Off.
- The app runs as a menu bar app.
- Login-at-startup is controlled by an explicit user setting.
- UI strings are localized in English and Japanese.
- The app is prepared for App Store distribution with sandbox and network entitlements.

## Current Implementation

This repository now contains a native macOS Xcode project generated by XcodeGen:

- `NSWorkspace.didWakeNotification`
- `NSWorkspace.willSleepNotification`
- `NSWorkspace.willPowerOffNotification`
- `NSWorkspace.extendPowerOff(by:)` for a short best-effort Power Off delay
- `SMAppService.mainApp` for the user-controlled login item setting

Power event monitoring starts automatically when the app launches. The Settings UI controls how each device reacts to those events; it does not enable or disable event monitoring itself.

The Matter control surface is isolated behind `DevicePowerControlling`. The current implementation stores multiple device records, applies per-device power-event actions, validates each configured Matter node/endpoint, records requests, and reports a visible unsupported-backend error until a commissioned Matter fabric or HomeKit bridge is wired to the selected device.

## HomeKit and Matter Architecture Note

The public HomeKit framework is not available to a native AppKit macOS target in the local macOS SDK. Apple DTS has described HomeKit on Mac as a Mac Catalyst capability. Because this app is required to be a native menu bar utility, it does not link the private macOS HomeKit framework.

The next implementation step should choose one of these releaseable paths:

1. Keep the native menu bar app and add a separate public-API bridge for Apple Home/HomeKit access.
2. Move the product to a Mac Catalyst app if direct HomeKit enumeration/control is the controlling requirement.
3. Implement a direct Matter controller with its own fabric, commissioning flow, persistent storage, CSA-assigned Vendor ID, and On/Off cluster control. This is separate from controlling Apple Home's existing fabric.

## Build

```sh
xcodegen generate
xcodebuild -project VirtConnector.xcodeproj -scheme VirtConnector -configuration Debug CODE_SIGNING_ALLOWED=NO build
```

For App Store distribution, configure the Apple Developer App ID `st.rio.virt-connector` with the required capabilities, select a valid development team in Xcode, and archive from Xcode.

## App Store Checklist

- Bundle ID: `st.rio.virt-connector`
- Category: Utilities
- License: MIT
- Privacy policy: `Docs/PRIVACY.md`; publish this content to a stable URL and register that URL in App Store Connect. The app itself does not expose an in-app privacy policy page.
- Login item: user-controlled through Settings only.
- Menu items: no keyboard shortcuts.
- Entitlements: sandbox and network client are enabled. Add HomeKit entitlement only to a public HomeKit-capable target, such as a Mac Catalyst bridge.
